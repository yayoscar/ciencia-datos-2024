
¿Qué hace el comando git init y en qué situaciones se utiliza? Este se utiliza para inicializar  un nuevo repositorio de git en un directorio cuando ejecutas este comando git crea un subdirectorio oculto llamado .git



¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git? Lo que hace git add es preparar los archivos y git commit confirma y guarda esos cambios en el historial


¿Que significa que un archivo este en estado "stage" en Git y como se coloca en este estado?Poner un archivo en estado stage significa que esta preparado para incluirse en el siguiente commit


¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard?  --soft: Revierte el commit pero mantiene los cambios en el área de preparación.
--mixed: Revierte el commit y mueve los cambios fuera del área de preparación.
--hard: Revierte el commit y descarta completamente los cambios realizados.


¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso? Una fusión (merge) en Git es el procedimiento mediante el cual se combinan dos ramas en una única, incorporando los cambios efectuados en cada una de ellas.

Conflictos de contenido: Suceden cuando dos ramas editan la misma línea de un archivo. Git no puede determinar automáticamente cuál versión mantener, por lo que requiere una corrección manual.
Conflictos de eliminación: Se producen cuando un archivo es eliminado en una rama pero modificado en otra, generando incertidumbre sobre su destino


¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch? git checkout y sus diferencias con git switch
git checkout: Permite cambiar de rama o restaurar archivos a un estado previo. Es más flexible, pero puede resultar confuso.
git switch: Una alternativa más reciente y específica para cambiar de rama, simplificando el proceso y haciéndolo más intuitivo.


¿Cómo funciona git pull y en qué se diferencia de git fetch? git pull contra git fetch
Ambos comandos permiten sincronizar el repositorio local con el remoto, pero tienen funciones distintas:

git fetch: Descarga los cambios del repositorio remoto sin aplicarlos a la rama local.
git pull: Descarga y fusiona automáticamente los cambios en la rama local, actualizándola al instante.


¿Qué ocurre cuando se ejecuta git push y qué problemas pueden surgir si hay cambios en el repositorio remoto? Al ejecutar git push, los cambios locales se envían al repositorio remoto. Sin embargo, si el remoto tiene modificaciones que no están en tu copia local, el push puede ser rechazado. Para solucionarlo, primero debes ejecutar git pull para actualizar tu repositorio y luego intentar el push nuevamente.


¿Cuál es la diferencia entre git clone y git init?La diferencia entre git clone y git init es:

git clone: Copia un repositorio remoto completo, incluyendo su historial y ramas, en tu máquina local.
git init: Crea un nuevo repositorio vacío en el directorio local, sin historial ni contenido, para comenzar un proyecto desde cero.
