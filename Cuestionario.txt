
¿Qué hace el comando git init y en qué situaciones se utiliza? 
Inicializa un nuevo repositorio de Git en el directorio actual, se usa cuando quieres empezar a usar Git en un proyecto.

¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git? 
Git add:Agrega cambios al área de preparación (staging area), preparando nuevos archivos para el siguiente commit.
Git commit:Guarda esos cambios en el historial del repositorio de manera permanente, junto con un mensaje descriptivo.

¿Que significa que un archivo este en estado "stage" en Git y como se coloca en este estado?
Es cando un archivo ha sido agregado al área de preparación Git add.Esto significa que está listo para ser incluido en el próximo commit.

¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard? 
-soft: Deshace el commit, pero mantiene los cambios en el área de preparación(staging)
-mixed: Deshace el commit y tambien saca los cambios del área de preparación, pero los mantiene en el área de trabajo.
-hard: Deshace el commit y elimina todos los cambios tanto del área de preparación como el directorio de trabajo.

¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso? 
Una fusión (merge)combina cambios diferentes ramas en una sola.Los conflictos que pueden ocurrir cuando dos ramas han modificado la misma línea de un archivo o han eliminado archivos que la otra rama aún usa.

¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch? 
git checkout: se usa para cambiar de rama o restaurar archivos a un estado anterior.
git switch: se usa para específicamente para cambiar de rama y es una alternativa más moderna y segura a git checkout.

 ¿cómo funciona git pull y en qué se diferencia de git fetch? 
git pull: Descarga los cambios del repositorio remoto y los funciona automáticamente con la rama actual.
hit fecht: Solo descarga los cambios sin fusionarlos , permitiendo revisarlos antes de aplicarlos.

¿Qué ocurre cuando se ejecuta git push y qué problemas pueden surgir si hay cambios en el repositorio remoto? 
git push sube los cambios locales al repositorio remoto. Si hay cambios en el remoto que no tienes en tu copia local, el push podría fallar y Git pedirá que primero hagas un git pull para sincronizar.

¿Cuál es la diferencia entre git clone y git init?
git clone: Crea una copia exacta de un repositorio reoto en tu máquuina.
git init: Crea un nuevo repositorio vacío en el directorio actual.

Flujo de trabajo en Git
Sigue un proceso estructurado para gestionar cambios en el código de manera eficiente.
