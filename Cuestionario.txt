¿Qué hace el comando git init y en qué situaciones se utiliza?
el git init hace un nuevo repositorio de git, crea una carpeta oculta donde se guardara todo el historial de versiones y Branch del proyecto.
se utiliza cuando se comienza un nuevo proyecto desde cero

¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git?
git add agrega archivos al área de preparación preparándolos para ser confirmados y git commit toma los archivos del área de preparación y los guarda en el historial de git 

¿Que significa que un archivo este en estado "stage" en Git y como se coloca en este estado?
significa que un archivo se preparara para ser confirmado 

¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard? 
soft: deshace el commit, pero mantiene los cambios en el área de staging
mixed: deshace el commit y quita los cambios del área de staging, pero los deja en el archivo
hard: deshace todo


¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso? 
es cuando se une los cambios de una rama con otra y los posibles conflictos que podrían ocurrir que los cambios no se puedan hacer automáticamente 


¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch?
git checkout sirve para moverse a una versión anterior de un archivo y git switch se usa para cambiar de rama

¿Cómo funciona git pull y en qué se diferencia de git fetch? git fetch descarga los cambios de un archivo y git pull descarga y funciona los cambios automaticamente
