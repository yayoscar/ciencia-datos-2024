
¿Qué hace el comando git init y en qué situaciones se utiliza? funciona para crear un nuevo repositorio 
 y una carpeta invisible que se llama .git y se usa para poder detectar las modificaciones que agamos en ella

¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git? git add prepara los archivos.
git add es para agregar archivos en la zona de preparación lamada (stage) y git commit confirma los cambio poniendo una firma o nota para subirlo en el repositorio


¿Que significa que un archivo este en estado "stage" en Git y como se coloca en este estado? en stage se usa para decir que el archivo ya listo para subirlo al commit, se coloca al crear un archivo en el repositorio como un .txt cuando ya este creado nos saldrá en el apartado de local changes estando ahí podremos ver el lado que se llama unstage donde originalmente estará, en la parte de arriba dira un botón stage al precionarle automáticamente se trasladara abajo donde dice stage y ya estara listo para subirlo

¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard?  
git reset funciona para eliminar los últimos commits y volver al estado anterior, 
-El git reset --hard borra tanto los commits posteriores como los archivos desde la computadora
-El git reset --mixed borra el commit y pone en archivo de ese commit borrado en "unstaged"
-Mientras que el git reset --soft deshace los commits pero mantiene los archivos en "staged"


¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso? Es una accion la cual se une una rama con la original (main)al hacer esto la otra rama traera los archivo que tiene en ella a el main un conflicto puede ser de contenido esto Ocurre cuando dos ramas modifican la misma línea de un archivo y el Git no puede decidir qué versión conservar, por lo que se necesita hacer de forma manual decidiendo que archivo se debe quedar puede ser el primero o el segundo o ambas 


¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch? El git checkout se usa para cambiar de rama de forma directa al crear la rama pero gir switch te cambia de rama pero no deja que al estar trabajando si tienes algún archivo pendiente no te dejara cambiar a otra rama


¿Cómo funciona git pull y en qué se diferencia de git fetch? git pull es para cuando se este trabajando en un repositorio en line como su dombre lo dice "jala" cambio que no hayan sido en el dispositivo y git fetch sirve para visualizar los cambios antes de hacer pull

¿Qué ocurre cuando se ejecuta git push y qué problemas pueden surgir si hay cambios en el repositorio remoto? en git push ocurre cuando se suben los cambios en el repositorio en linea, el conflicto que puede generarse puede ser iguales a los deñ git merge cuando hay varias versiones del mismo archivo o otros cambios en el repositorio en linea

¿Cuál es la diferencia entre git clone y git init?git clone es para copiar un repositorio en linea copiando la URL para trasladar lo archivos en el dispositivo y poder trabajar en el y git init crea un nuevo repositorio para trabajar en ella directamente en el explorador del dispositivo y no en linea
