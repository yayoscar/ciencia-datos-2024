¿Qué hace el comando git init y en qué situaciones se utiliza?
Se crea un repositorio git o un proyecto ya existente y tiene como proposito usarse una vez durante la configuracion inicia.
¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git?
el comando git add para seleccionar los cambios que se prepararán para la siguiente confirmación y git commit para crear una instantánea de los cambios preparados en un cronograma de un historial de proyectos de Git.
¿Que significa que un archivo este en estado "stage" en Git y como se coloca en este estado?
Cuando un archivo esta en stage esta en preparacion para ser confirmado en nuestras actualizaciones o movimientos. 
¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard?
Se utiliza para restablecer git a su estado inicial donde los cambios se conservan o se desechan soft -> prácticamente sin cambios. mixed -> limpia la staging area. hard -> elimina todos los cambios y vuelve alineados con HEAD.
¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso? 
una fusion es la combinacion entre ramas que se forman en una sola y los conflictos passan cuando hay cambios en mismo archivo desde diferentes ramas y esto pasa cuando alguien edita y/o elimina otra persona lo elimina.
¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch? 
Git checkout se usa para cambair de rama, actualizar archivos, restaurar y recuperar los estados de cada archivo pero 
git switch solo cambia de rama.
¿Cómo funciona git pull y en qué se diferencia de git fetch? 
Si bien los dos sirven para ser cambios y actualizar desde los codigos locales sin embargo pull funciona automaticamente y fetch no.
¿que ocurre cuando se ejecuta git push y que problemas pueden surgir si hay cambios en el repositorio remoto?
se rechaza los cambios que se intenten escribir en el cambio local.

Git int solo se crea un nuevo repositorio y git clone solo clona un repositorio ya existente.
