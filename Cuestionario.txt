
¿Qué hace el comando git init y en qué situaciones se utiliza? R=el comando git init e utiliza para crear un nuevo repositorio en un directorio existente, se usa para crear un nuevo proyecto/repositorio o iniciar uno desde uno ya existente


¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git? R=el comando git add se utiliza para agregar cambios al área de preparación, toma los cambios realizados en el directorio de trabajo y los copia al área de preparación. Por otro lado, el comando git commit se utiliza para confirmar los cambios pasados de la zona "unstage" a "stage".


¿Qué significa que un archivo esté en estado "staged" en Git y cómo se coloca en este estado? R= el estado stage de un archivo se refiere a que fue previamente padado de la zona unstage a stage, y significa que esta a punto de confirmarse ese archivo o cambio


¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard? R= el git reset --soft,e utiliza para deshacer el commit mas reciente sin afectara el directorio ni el área de preparación. Por otro lado, el git reset --mixed se utiliza para deshacer el commit más reciente y los cambios en el área de preparación. Por último, el comando git reset --hard se utiliza para deshacer commits y cambios del área de preparación y deshace los cambios en el directorio de trabajo.


¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso? R= Merge significa fusionar la rama creada con la main/master, pueden haber conflictos por varias razones, entre ellas, por el contenido (hubo dos modificaciones a la misma línea de código) o cuando un archivo a sido renombrado en una rama y modificado en otra.


¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch? R= el comando checkout se utiliza para cambiar entre ramas, pero puedes hacer distintas cosas como revertir cambios o restaurarlos, a diferencia del comando switch que solo sirve para cambiar entre ramas.


¿Cómo funciona git pull y en qué se diferencia de git fetch? R= el git pull se utiliza para descargar los cambios realizados en el repositorio remoto y aplicarlos al local. Por otro lado el git fetch solo descarga los cambios realizados en el repositorio remoto.


¿Qué ocurre cuando se ejecuta git push y qué problemas pueden surgir si hay cambios en el repositorio remoto?
R= el comando git push se utiliza para enviar los cambios realizados en el repositorio local al remoto, pero pueden haber problemas si el repositorio remoto ha sido actualizado con cambios que no están en el local, o si el repositorio remoto ha sido actualizado con cambios que están en conflicto con los cambios realizados en el repositorio local.
