¿Qué hace el comando git init y en qué situaciones se utiliza?
El comando git init es fundamental para establecer un nuevo repositorio de Git. Se utiliza principalmente en dos situaciones: para crear un repositorio desde cero o para convertir un proyecto existente.

¿Cuál es la diferencia entre git add y git commit en el flujo de trabajo de Git?  
Git add se utiliza para seleccionar los cambios en los archivos que se desean preparar para la próxima confirmación y git commit para crear una instantánea de esos cambios y registrarlos en el historial del proyecto. 

¿Que significa que un archivo este en estado "stage" en Git y como se coloca en este estado?
Se refiere a la preparación de archivos para ser incluidos en el próximo commit. Para colocar un archivo en este estado, se utiliza el comando git add

¿Cuál es la diferencia entre git reset --soft, git reset --mixed y git reset --hard?
soft -> prácticamente sin cambios. mixed -> limpia la staging area. hard -> elimina todos los cambios 

¿Qué es una fusión (merge) en Git y cuáles son los posibles conflictos que pueden ocurrir durante el proceso?
Se realiza con el comando git merge, que permite tomar diferentes líneas de desarrollo y combinarlas en una sola, los cuales ocurren cuando hay cambios incompatibles en el mismo archivo desde diferentes ramas. 

¿Para qué se usa el comando git checkout y cuál es su diferencia con git switch? 
Git checkout se utiliza tanto para cambiar ramas como para restaurar estados de archivos, mientras que git switch está diseñado específicamente para cambiar ramas.

¿Cómo funciona git pull y en qué se diferencia de git fetch? 
Mientras que git fetch se encarga de recuperar los cambios del repositorio remoto sin aplicarlos a la rama local, git pull combina esta acción con un merge, integrando automáticamente esos cambios en la rama activa.

¿Qué ocurre cuando se ejecuta git push y qué problemas pueden surgir si hay cambios en el repositorio remoto?
Git push se utiliza para subir los cambios de un repositorio local a un repositorio remoto, si existen cambios en el repositorio remoto, Git no nos dejará subir cambios al repositorio hasta que hayamos descargado la versión más reciente y la hayamos fusionado con nuestra versión local.

¿Cuál es la diferencia entre git clone y git init?
Git init se utiliza para iniciar un nuevo repositorio de Git en un directorio vacío o existente y git clone se utiliza para crear una copia de un repositorio existente.
